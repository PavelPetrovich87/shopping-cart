## 1. Ubiquitous Language
- **Cart:** A container for items where we can add, remove, and change the quantity of things.
- **Cart Item:** A representation of the Inventory Item we are currently working with inside the Cart.
- **skuId:** The unique identifier representing a specific variant of an inventory item.

## 2. Aggregate Root
**Cart**
- **Description:** Manages the user's active shopping cart, items, and calculating the raw subtotal.
- **Invariants:**
  - A `CartItem`'s quantity cannot be less than 1.
  - A `CartItem`'s quantity cannot exceed available stock.
  - A `Cart` can contain multiple coupons.
  - `CartItem` entities are uniquely identified by `SkuId` within a Cart.

## 3. Structural Elements
### Entities
- `CartItem` (Identifier: `skuId`): Re-represents an item in the cart whose properties (like quantity) can change.

### Value Objects
- `Money`: Represents financial amounts avoiding floating-point precision issues.

## 4. Lifecycle
- States: `Active` -> `Checkout_Pending` -> `Checked_Out`
- Transitions: 
  - `InitiateCheckout` triggers `Active` -> `Checkout_Pending`
  - `CartCheckedOut` triggers `Checkout_Pending` -> `Checked_Out`

## 5. Ports (CQRS)
### Driving Ports (Inbound)
#### Commands (Mutations)
- `AddItemToCart`: Adds a new item or increases existing item quantity.
- `RemoveItemFromCart`: Removes an item entirely from the cart.
- `ChangeCartItemQuantity`: Updates the quantity of a specific item.
- `ApplyCouponToCart`: Adds a discount coupon to the cart.
- `RemoveCouponFromCart`: Takes off a previously applied coupon.
- `InitiateCheckout`: Starts the checkout process and transitions the state.
#### Queries (Reads)
- `GetCart`: Retrieves the current state and subtotal of the cart.

### Driven Ports (Outbound)
- `IInventoryService`: Queries stock availability synchronously before adding/changing quantities.
- `IPricingService`: Validates code and calculates the actual order discount.

## 6. Domain Events
- `ItemAddedToCart`: Triggered when an item is added to the cart.
- `CartItemQuantityChanged`: Triggered when item quantity is modified.
- `ItemRemovedFromCart`: Triggered when an item is removed.
- `CouponApplied`: Triggered when a coupon is successfully applied.
- `CouponRemoved`: Triggered when a coupon is removed.
- `CartCleared`: Triggered if the cart is fully emptied.
- `CheckoutInitiated`: Triggered to tell Inventory (and others) that checkout has started.
- `CartCheckedOut`: Triggered when the checkout process completes successfully.

## 7. Context Relationships
- **Inventory Context:** Upstream - Provides synchronous stock availability validation.
- **Pricing & Promotions Context:** Upstream - Provides synchronous coupon validation and discount calculation.
