## 1. Ubiquitous Language
- **ProductVariant:** The specific version of a product (e.g., Orange vs. Pink) being stocked.
- **Stock Validation / Availability:** Checking if we have enough of a variant on hand to fulfill a request.
- **Reservation:** Temporarily holding a specific quantity of stock for a particular order that has initiated checkout.
- **Depletion:** Permanently removing stock once an order is fully confirmed and payment is successful.

## 2. Aggregate Root
**ProductVariant**
- **Description:** Manages product variants, available stock levels, and active reservations.
- **Invariants:**
  - Stock `TotalOnHand` cannot be negative.
  - Must emit specific events (`StockReserved`, `StockDepleted`) when stock status changes so other contexts are aware.

## 3. Structural Elements
### Entities
- `ProductVariant` (Identifier: `skuId`): The main entity holding stock totals and the ledger of active reservations.

### Value Objects
- `StockReservation`: An immutable record representing a hold on inventory (Fields: `orderId`, `quantity`, `reservedAtTimestamp`).

## 5. Ports (CQRS)
### Driving Ports (Inbound)
#### Commands (Mutations)
- `ReserveStock`: Creates `StockReservation` objects for given items based on an order.
- `ReleaseStockReservation`: Removes the `StockReservation` objects for an order (e.g., on checkout timeout).
- `ConfirmStockDepletion`: Removes the `StockReservation` AND reduces the `TotalOnHand` permanently.

#### Queries (Reads)
- `CheckStockAvailability`: Calculates availability (`TotalOnHand` - total reserved quantities) to tell callers if the item can be pulled.
- `GetProductVariant`: Returns the current state of a variant.

### Driven Ports (Outbound)

## 6. Domain Events
- `StockReserved`: Triggered when stock is successfully held for a specific order.
- `StockDepleted`: Triggered when stock is permanently removed after a confirmed order.
