## Technical Standards

### Architecture and Design Patterns
- **Domain-Driven Design (DDD)**: Must follow DDD approaches strictly.
- **Hexagonal Architecture**: Must use ports and adapters pattern to separate domain from infrastructure and user interface.
- **Learning Project**: Act as a learning ground applied to DDD and hexagonal patterns on the frontend.
- **State vs. Events (Zustand + Event Bus)**:
  - **State Management (Zustand)**: Strictly used as an Infrastructure Read Model / Repository adapter to hold the current application state for the React UI.
  - **Domain Events**: Generated by pure Domain Entities as records of what happened, returned alongside the updated entity state (e.g., `{ updatedCart, events }`).
  - **Application Service Orchestration**: Application Use Cases must coordinate both. They read state from the repository (Zustand), execute the pure domain logic, save the updated state back to the repository, and explicitly publish the resulting Domain Events using a dedicated, separate `EventBus` port. Domain Entities MUST NOT interact with the EventBus or State manager directly.

### Languages and Frameworks
- **React 18** with **TypeScript 5.x**, scaffolded via **Vite**.
- **Zustand** for state management / repository layer (driven adapters).
- **Vanilla CSS** (responsive, mobile-first). No TailwindCSS unless explicitly requested.
- **Vitest + React Testing Library** for automated testing.
- Semantic HTML and ARIA roles for web accessibility.
- Optimized CSS and JavaScript for fast load times.

### Testing Requirements
- Code implementations must be thoroughly tested according to the user's explicit specifications later on.
- Real-time stock validations, quantities calculations, and subtotal/total calculations must be bulletproofed against edge cases.

### Performance and Scale
- Perform real-time evaluations: Subtotal, shipping, and total costs must calculate automatically and accurately.
- Cross-browser capability: Must work flawlessly on Chrome, Firefox, and Safari.
- Responsive design adaptation scaling from mobile devices to desktop computers accurately matching provided reference designs.

### Deployment and Constraints
- Hosting on modern edge platforms like GitHub Pages, Vercel, or Netlify.
- **No backend for now.** All data mocked via in-memory repositories. Architecture must use Repository ports so real API adapters can be swapped in later without touching Domain or Application layers.
